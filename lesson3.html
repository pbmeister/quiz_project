Lesson 3: XSS (Basic)

XSS happens when attackers insert malicious scripts into web pages that others see. These scripts can steal cookies, hijack sessions, or change content. Proper input/output handling and sanitization prevent XSS attacks. Cross-Site Scripting (XSS) happens when attackers inject malicious scripts into pages viewed by other users. This can steal session cookies, redirect users, or display unwanted content. Escaping or encoding user-generated content before displaying it prevents scripts from executing in the browser.
Takeaway: Always sanitize output, not just input. Escaping user content protects users from malicious scripts.
